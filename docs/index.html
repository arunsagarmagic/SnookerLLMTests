🎱 Snooker LLM Tests Results

Collection of Snooker Scoring webapp implementations generated by different LLMs. Each result shows a complete application created from a single prompt.

A variety of LLM models were tested with a single prompt to establish a checkpoint of performance. Scoring is coarse grain (A-F).

Existing terminology for language model size is:
SLM <= 10B, 10B < MLM <=100B, LLM> 100B.

Given that we have 1T and larger models, it is useful to extend the terminology: XL <=1T, XXL <=10T, etc.

Prompt Used

Build a snooker scoring webapp. Use indexdb for storage, with clean database APIs so that a normal database can be plugged later. The app should take the two player's name and then provide an intuitive UI for entering the current state of the game by showing the colored balls (3d/shaded). It should also show a running score on same page prominently. It'll be used as the game is played. The history should be tracked.
